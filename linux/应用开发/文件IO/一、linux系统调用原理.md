# 一、计算机组成
**1.硬件系统：** 控制器、运算器、存储器（RAM，ROM）、输入设备、输出设备
  **机器字长=数据总线宽度！=地址总线位数**(只取决于寻址空间的大小)
**2.软件系统：** 用户应用，系统应用
**3.文件系统：** 文件存储的标准（linux常用：ext4、XFS、btrfs Windows常用：FAT32，NTFS，exFAT、ReFS）
**文件系统=硬盘管理规则+索引表**—>告诉你文件存在哪里，占多大，叫什么，怎么读
==（**补充**：了解常见文件系统的区别）==
|文件系统|主流适用系统|单个文件最大|分区最大容量|兼容性|日志功能|核心特点|推荐使用场景|
|:--|:--|:--|:--|:--|:--|:--|:--|
|FAT32|全平台通用|4GB|2TB|所有系统、设备均支持|无|兼容性极强，无权限 / 加密|小容量 U 盘、老式设备、跨平台小文件|
|exFAT|Windows/macOS/Linux/ 智能设备|16EB|128PB|跨平台优秀，无 4GB 限制|无|轻量、大文件友好|大容量移动硬盘、跨设备大文件传输|
|NTFS|Windows（默认）|16EB|16EB|Windows 原生；macOS 只读；Linux 需驱动|有|权限、加密、压缩、稳定|Windows 系统盘、本地内置硬盘|
|ext4|Linux（默认）|16TB|1EB|Linux 原生；Win/macOS 需工具|有|高效稳定、日志、性能均衡|Linux 系统盘、服务器、嵌入式存储|
|APFS|macOS/iOS|8EB|8EB|苹果设备原生|有|快照、加密、空间共享|Mac 系统盘、苹果设备内置存储|
|XFS|Linux 服务器|8EB|16EB|Linux 专用|有|大文件 / 高并发性能强|数据库、大容量存储服务器|
