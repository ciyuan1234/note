## **3.文件系统**
文件存储的标准（linux常用：ext4、XFS、btrfs Windows常用：FAT32，NTFS，exFAT、ReFS）
**文件系统=硬盘管理规则+索引表**—>告诉你文件存在哪里，占多大，叫什么，怎么读
### ==**补充**：常见文件系统的区别==
1. FAT32
- 最老、兼容性**最强**
- 几乎所有设备都能读：Windows、macOS、Linux、U 盘、电视、相机、PS、Xbox
- 缺点：
- 单个文件**不能超过 4GB**
- 分区最大 2TB
 - 适合：U 盘、移动硬盘跨设备通用
1. exFAT
- FAT32 的升级版
- 无 4GB 限制，支持超大文件
- 跨平台兼容性依然很好：Windows /macOS/ Linux / 电视 / 相机
- 缺点：没有日志，意外断电容易丢数据
- 适合：大容量移动硬盘、跨平台大文件传输
3. NTFS（Windows 默认）
- Windows 系统盘标配
- 支持大文件、大分区、权限、压缩、加密、日志
- 稳定性强，不容易损坏
- 缺点：
- macOS 只能读不能写（需装插件）
- Linux 支持一般
- 电视 / 游戏机大多不识别
- 适合：Windows 系统盘、内部硬盘
1. APFS（macOS 默认）
- 苹果现代文件系统
- 速度快、快照、空间共享、加密强
- 只适合 macOS /iOS
- Windows、Linux 基本不能读写
- 适合：Mac 系统盘、苹果设备
5. ext4（Linux 默认）
- Linux 最主流
- 稳定、高效、支持大文件、日志、权限
- Windows 几乎不支持，macOS 需工具
- 适合：Linux 系统盘、服务器
6. Btrfs / XFS（Linux 高级）
- Btrfs：支持快照、校验、自动修复，适合桌面 / 存储
- XFS：大文件、高并发，适合服务器 / 数据库
- 普通用户不用关心

## **2.根文件系统**
**定义：** Linux中挂载在根目录下的文件系统，是Linux内核挂载的第一个文件系统，是Linux的唯一顶层目录，属于文件系统的一种（除此以外，还有其他的文件系统，但都需要挂载在跟文件系统）
（和操作系统内核是合作关系，并不归属于Linux内核）
![](../../../assets/Pasted%20image%2020260328212710.png)
  ==**2.1Linux核心思想：**==**一切皆文件**（Linux把脚本、文档、硬件设备都抽象成了文件）
  **2.2Linux有7种文件类型：** 
1) 普通文件（- **regular**） ：存在于外部存储器中，用于存储普通数据。
2) 目录文件（d **directory**） ：用于存放目录项，是文件系统管理的重要文件类型。
3) 管道文件（p **pipe**） ：一种用于进程间通信的特殊文件，也称为命名管道FIFO。
4) 套接字文件 （s **socket**）：一种用于网络间通信的特殊文件。
5) 链接文件 （l **link**）：用于间接访问另外一个目标文件，相当于windows快捷方式。
6) 字符设备文件（c **character**）：字符设备在应用层的访问接口。
7) 块设备文件 （b **block**）：块设备在应用层的访问接口。
  **2.3linux中目录不是文件夹，它只记录文件信息，作为索引文件，不作为容器存储**
